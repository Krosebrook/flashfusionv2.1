## Breaking Changes Policy

We follow semantic versioning:
- **Major versions** (3.0.0): Breaking changes allowed
- **Minor versions** (2.x.0): New features, backward compatible
- **Patch versions** (2.1.x): Bug fixes, backward compatible

Breaking changes will be:
- Announced at least 3 months in advance
- Documented with migration guides
- Supported with deprecation warnings
